# v0.5.3 AbortSignal Threading Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close v0.5.2 spec §5 "concurrent retry storm" — thread `AbortSignal` through `LLMProvider.chat` so per-attempt timeouts actually abort the underlying HTTP request, bounding token cost to 1× sequential instead of 4× concurrent for Claude/OpenAI/Custom (Google bounded to 1× per `/councildone` via hard-fail on first timeout).

**Architecture:** Add optional `signal: AbortSignal` to `ChatOptions` (decision 1B). Wire 4 providers to forward `signal` to their SDKs natively. Replace `Promise.race` body in `invokeProviderForArtifact` and the `raceWithTimeout` callsite for facilitator intervention with race+signal-in-parallel pattern (signal best-effort, race deterministic backstop — round-3 invariant). `AgentWorker.respond()` gains optional 7th positional `signal?` (additive; only facilitator-path passes a non-undefined value). Google's `abortSignal` is client-side only per `@google/genai` docs, so first Google timeout throws `GoogleProviderTimeoutError` (subclass of `ProviderTimeoutError` carrying user-facing disclosure) and skips retry.

**Tech Stack:** TypeScript (Node ESM, `>=22`), `vitest` for tests, existing 4 LLMProvider implementations (`@anthropic-ai/sdk`, `openai`, `@google/genai 1.49`, raw fetch), `AbortSignal.any` (Node 20.3+ / TS lib ES2024).

**Spec:** [`docs/superpowers/specs/2026-04-29-v0.5.3-abortsignal-threading-design.md`](../specs/2026-04-29-v0.5.3-abortsignal-threading-design.md) — 5 codex review rounds, 0 P1 ship gate.

## Pre-Flight: Spec Resolved Items

- **§3.1 / §3.2** — `signal` lives in `ChatOptions`; `AbortSignal.any` for merge; `TimeoutReason` carried on `signal.reason`
- **§0.2 option C** — Google: first timeout = hard fail, no retry, throw `GoogleProviderTimeoutError`
- **§0.3 invariant** — every existing in-scope timeout site keeps `Promise.race` deterministic backstop AND adds signal in parallel
- **§5.1 site 5 single-timer** — one `setTimeout` does both race-reject and signal-abort; `reject()` runs synchronously before `abort()`
- **§6.1.1** — `GoogleProviderTimeoutError extends ProviderTimeoutError`; user message baked into class; adapters need zero changes
- **§8 file location** — `src/abort-utils.ts` at top level (NOT under `src/council/`)

Out-of-scope deferrals (do NOT implement here, all v0.5.4+):

- User-initiated `/councilreset` / `/councildone` cancellation
- Thread-scoped `AbortController` map on `DeliberationHandler`
- Peer-turn `respond()` callers passing signal
- `respondDeterministic()` signature change (used only by SessionReset; v0.5.4)
- IntentGate / memory layer / dispatcher cancellation
- `provider.summarize()` signature change

## Architectural Conventions

**Conditional spread for optional ChatOptions fields:** existing `provider.chat` calls already use `...(thinking && { thinking })` and `...(systemPromptParts && { systemPromptParts })` (see `agent-worker.ts:104-108`). Match this for `signal`.

**Test layout:** `tests/worker/providers/` and `tests/council/` exist. Don't invent `tests/unit/...` paths.

**Error class style:** existing classes in `artifact-errors.ts` extend `Error` and set `this.name`. `GoogleProviderTimeoutError` extends `ProviderTimeoutError` and overrides `message` in constructor.

**Provider name field:** every `LLMProvider` impl has `readonly name: string` per `base.ts:4`. Values: `'claude'`, `'openai'`, `'google'`, `'custom'`.

export function isAbortError(err: unknown): boolean {
  if (!err || typeof err !== 'object') return false;
  const e = err as { name?: string; code?: string; constructor?: { name?: string } };
  if (e.name === 'AbortError') return true;
  if (e.code === 'ABORT_ERR') return true;
  if (e.constructor?.name === 'APIUserAbortError') return true;
  return false;
}
```

export function mergeSignals(
  ...signals: (AbortSignal | undefined)[]
): AbortSignal | undefined {
  const present = signals.filter((s): s is AbortSignal => s !== undefined);
  if (present.length === 0) return undefined;
  if (present.length === 1) return present[0];
  return AbortSignal.any(present);
}

export function isTimeoutAbort(signal: AbortSignal): boolean {
  return signal.aborted && signal.reason instanceof TimeoutReason;
}
```

export async function invokeProviderForArtifact(
  provider: LLMProvider,
  messages: ProviderMessage[],
  options: ChatOptions,
  perAttemptTimeoutMs: number = PER_ATTEMPT_TIMEOUT_MS,
): Promise<ProviderResponse> {
  const timeoutCtrl = new AbortController();
  const merged = mergeSignals(options.signal, timeoutCtrl.signal);

  let timeoutId: ReturnType<typeof setTimeout> | undefined;
  try {
    return await Promise.race([
      provider.chat(messages, { ...options, signal: merged }),
      new Promise<never>((_, reject) => {
        timeoutId = setTimeout(() => {
          timeoutCtrl.abort(new TimeoutReason(perAttemptTimeoutMs));
          reject(new ProviderTimeoutError(perAttemptTimeoutMs));
        }, perAttemptTimeoutMs);
      }),
    ]);
  } catch (err) {
    if (err instanceof TimeoutReason) {
      throw new ProviderTimeoutError(perAttemptTimeoutMs);
    }
    if (err instanceof ProviderTimeoutError) {
      throw err;
    }
    if (isAbortError(err)) {
      if (merged?.aborted && merged.reason instanceof TimeoutReason) {
        throw new ProviderTimeoutError(perAttemptTimeoutMs);
      }
      throw err;
    }
    throw err;
  } finally {
    if (timeoutId !== undefined) clearTimeout(timeoutId);
  }
}
